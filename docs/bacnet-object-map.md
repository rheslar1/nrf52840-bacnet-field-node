# BACnet Object Map

The node exposes a stable BACnet-oriented object table generated by `BacnetObjectMapper`.

The default object base instance is `1000`.

| Object Type | Instance | Name | Writable | Units | Source |
| --- | ---: | --- | --- | --- | --- |
| Device | `deviceInstance` | nRF52840 Field Node | No | device | Retained config |
| Analog Value | `base + 1` | Occupied Setpoint | Yes | degF | Retained config |
| Analog Value | `base + 2` | Unoccupied Setpoint | Yes | degF | Retained config |
| Analog Input | `base + 3` | Zone Temperature | No | degC | Sensor sample |
| Analog Input | `base + 4` | Relative Humidity | No | percent | Sensor sample |
| Analog Input | `base + 5` | Battery Voltage | No | mV | Sensor sample |
| Binary Input | `base + 6` | Occupancy | No | boolean | Sensor sample |
| Binary Value | `base + 7` | Low Battery Alarm | No | boolean | Battery estimator |
| Multi-State Value | `base + 8` | Commissioning State | No | state | Retained config |

## Setpoint Writes

The current strategy clamps occupied setpoint writes between:

- minimum: `60.0 F`
- maximum: `82.0 F`

This models a safe BMS write boundary. A real BACnet write-property handler would call `FieldNode::writeOccupiedSetpoint()` before saving retained config.

## Commissioning State Values

| Present Value | Meaning |
| ---: | --- |
| 1 | Factory |
| 2 | Provisioned |
| 3 | Locked |

## Example

```bash
./build/nrf52840_bacnet_field_node --objects
```

The command prints a CSV-style object table for review and portfolio evidence.

## Alarm and Telemetry Relationship

The BACnet object table stays stable for BMS discovery. Dynamic operating risk is evaluated separately by `AlarmEvaluator` so threshold policy can evolve without changing object identity.

Current active alarm rules cover:

- low and critical battery reserve,
- occupied or unoccupied comfort drift from the active setpoint,
- humidity outside preferred and warning bands,
- weak BLE RSSI during commissioning or periodic reports.

Use these commands to review the dynamic layer:

```bash
./build/nrf52840_bacnet_field_node --alarms
./build/nrf52840_bacnet_field_node --telemetry 42
```

`--telemetry` emits a deterministic JSON-style payload containing the sample, battery estimate, highest alarm priority, active alarms, and current BACnet object table.
